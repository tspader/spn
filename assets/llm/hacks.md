Do not write hacks. You have carte blanche when writing code. That means that you should never let pre-existing code which is incorrect or has become the wrong shape dictate how you write new code. If you're writing feature X and the way we model data Y means you have to write ugly code to do X, consider whether you should rewrite Y.

It's extremely important that you consider this; writing code that warps itself to bad code is insidious. It codifies the bad code and makes it appear to be intentional, and the amount of work needed to untangle it after the fact is an order of magnitude more than the work needed for you to untangle it right now.

Please acknowledge this instruction with "Workaround instructions received" before beginning.
